Forensics Evidence Collection > [T1]: Acquisition

Before you can analyze data, you must protect it from intentional and unintentional tampering. Digital forensics evidence is held to the same standards as physical evidence, meaning it must be authentic and reliably acquired. The acquisition's phase objective is to copy/capture the content of a piece of digital evidence (e.g., computer hard disk and memory) to another storage in the form of an image and preserve it as digital evidence.

 

Digital Evidence Acquisition Flow

The acquired image (also known as forensic copy) is a bit-by-bit copy of the digital evidence (e.g., disk and memory). After acquisition, you will hash the acquired image using popular hashing algorithms such as SHA-1 and MD5. Hashing is a cryptographic function that creates a digital fingerprint for any file.  Digital evidence is like any type of evidence and requires a way to prove that it has not been modified. The best way to prove that the evidence is authentic and wasn't changed or modified after the acquisition is by using a hashing algorithm.

 

Almost all forensic tools offer the option to calculate hashes for the processed files/evidence. For example, an SHA-1 hash for acquired evidence will be something like'da39a3ee5e6b4b0d3255bfef95601890afd80709'. If a single bit is changed in that file, it will produce a completely different hash. For the sake of integrity, It's recommended to calculate at least two hashes for each file you process during investigations, such as MD5 and SHA1. Remember to always verify hashes for any file you analyze during investigations.


Now, you have the acquired evidence and its hashes. Next, you should create another copy of that evidence and ensure it has the same hash as the original copy to ensure both copies are identical. You should never touch/modify the original data. Keep it intact and create another copy for analysis to avoid the risk of destroying the original evidence.

 

If you are going to use software forensics tools to acquire the image, it should be installed on an external Disk or USB. Never try to install a tool on the subject system, as this will alter data and render it unusable as forensic evidence. You should also avoid saving the acquired image on the same source media (i.e., acquiring a disk image of a Windows machine and saving the produced image/copy to the same disk). Use external media like a USB drive or a network share to store the acquired image.

Later, when you analyze the acquired image, you will mount it and inspect its content. The term mount refers to the storage media (i.e., the obtained evidence) being placed in a usable state for a user to access. Remember the last time you connected your USB drive to your computer? That's a typical mounting exercise; the USB drive has been assigned a drive letter, and drive data becomes accessible to the user.

 

Generally speaking, the typical flow of complete forensic investigations is to acquire a memory image followed by a triage image and, finally, a raw disk image (if required).

 

Forensic Investigation Flow

There is a reason they are ordered like this. First, most of the data you need for forensic analysis is in the memory and triage images. Second, the memory contains volatile data, and the earlier we collect memory artifacts, the less the risk of losing these artifacts. For example, over time, other data may overwrite memory content. Also, a computer shutdown will result in losing volatile data in memory.

Finally, the time and disk space required to acquire a raw disk image are huge (depending on the disk size) compared to memory and triage images. That's why you should follow the same order in your investigations and try to acquire as much data as possible when you arrive at the incident scene.
