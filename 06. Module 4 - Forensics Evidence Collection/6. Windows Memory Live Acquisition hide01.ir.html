Forensics Evidence Collection > [T2]: Windows Memory Live Acquisition

Windows Systems

Many tools can acquire a memory image on a live Windows system, producing an image file you can analyze later using a set of forensic analysis tools. Still, there are multiple factors to consider when choosing which tool to use:

  1. Footprint: Like any other tool/program, the acquisition tool running on the windows operating system will use part of the computer memory. Hence, it may overwrite parts of the memory containing important attack artifacts. The more memory the acquisition tool needs to operate, the higher the likelihood of overwriting significant evidence.
  2. Mode: Acquisition tools can operate in one of two modes; User-Mode & Kernel-Mode. Since kernel mode runs deeper than user mode, tools that operate in kernel mode can access and capture memory parts inaccessible to user-mode tools. In addition, kernel-mode tools can bypass anti-debugging and anti-dumping protection, standard techniques attackers use to make it harder for defenders to analyze the attack and trace back the attacker's actions.
  3. Speed: as every second pass, the memory content keeps changing. Some malware may perform cleanup routines. Thus, acquiring a memory image as fast as possible is always preferred.
     
Dumpit, BelkaSoft, and FTK Imager are some of the most popular memory acquisition tools.
