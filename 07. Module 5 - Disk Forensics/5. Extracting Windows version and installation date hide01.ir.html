Disk Forensics > [T2]: Extracting Windows version and installation date

(1) Windows version and installation date

The primary information to collect are version, service pack, build number, and release ID. You will find all of that under the following registry key:

SOFTWARE\Microsoft\Windows NT\CurrentVersion

 

CurrentVersion" registry key (zoom-in) 

  1. Current Version registry key.
  2. The full path of the key.
  3. Current build: Windows build number.
  4. Current version: Windows version. Refer to this comprehensive list for version mapping.
  5. Installation type
  6. Product name: full name of the windows edition (Windows 10 Enterprise).
  7. Installation date: in rare situations, insiders may reinstall OS to cover their tracks. Identifying when the OS we installed gives you an indication of how far you can go back. Note that both 'InstallTime' and 'InstallDate'  store the same value but in different formats. The first uses Windows FILETIME timestamp format, and the latter uses Unix Timestamp format.

You can decode both fields in 'RegistryExplorer' by right-clicking on the value and choosing 'Data interpreter'.

 

Extracting information from the registry can work on both live and offline systems. However, if you need a quick way to get this information on a live system, you can run the 'winver' command.
