(3) Network shares

Sharing files and folders is something common in an enterprise network. Attackers use compromised credentials to scan and access available network shares, hoping to find something valuable (e.g., administrative credentials, API keys, trade secrets...etc.). 

Windows stores shared objects' data under SYSTEM\ControlSet001\Services\LanmanServer\Shares key.

 

Shares key - (zoom in)

 

  1. "Shares" registry key.
  2. "Shares" registry key path.
  3. A list of shared objects, one for each share. Two shares in the above example; 'users' and 'secrets'. Clicking any of them will display share details in the bottom section (4).
  4. Share details such as 
    • Path: the local path of the shared object.
    • Permission: how the share was created? '0' for simple sharing GUI, '9' for advanced sharing GUI, and '63' 'for cmd line created shares.
    • 'ShareName': the share name on the network, regardless of the actual local file/folder name.
    • Type: '0' means a drive or folder, '1' implies a printer, and '2' indicates a device.

If you did not find any values under the 'Shares' key, then no shares have been created.
