Disk Forensics > [T4]: Exploring the Security Account Manager

(1) Security Account Manager (SAM)

Security Account Manager (SAM) registry hive stores most of the user's data you will need. The OS locks the SAM file located under C:\Windows\System32\config\SAM from reading/copying on a live running system. However, there are multiple ways you can obtain it if you have administrative privileges. You can look for a backup SAM file at C:\Windows\Repair\SAM or dump it from a privileged CMD by executing the following command: reg.exe save hklm\sam C:\temp\sam.dump.

Because of its importance, the SAM file is a popular target for attackers looking after users' passwords. Attackers utilize third-party tools to dump it, like secretsdumpy.py and CrackMapExec.


In a forensic investigation context, you probably will have a disk or triage image (i.e., a non-live system), and you will not face the above situation. You can use "RegRipper" to parse the SAM registry hive and export it as a text file.

Exported SAM.txt - (zoom in)

     1. Username: Account's username (IE User).
     2. Account Created: Account's creation date.
     3. Last Login Date: last time the user logged in.
     4. Pwd Reset Date: last time the account's password changed.
     5. Login Count: how many times this account logged into the system.
     6. Embedded RID: relative identifier of the account. The Relative ID (RID) is the last part of a SID (1000).
     7. Users field contains the account SID. A user SID consists of two parts, the first part is the machine SID (S-1-5-21-321011808-3761883066-353627080), and the second part is the account's RID (1000).
