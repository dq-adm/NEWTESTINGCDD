In this lesson, we will discuss the key components of the Elastic Stack and the architecture that underpins Elastic SIEM.

Elastic Stack Components

Elastic Stack, formerly known as ELK Stack, consists of three primary components:

  1. Elasticsearch (E): A powerful open-source search and analytics engine that enables data indexing and analytics. Elasticsearch forms the core of Elastic SIEM, providing fast search capabilities and scalability.

  2. Logstash (L): A data shipping and transformation tool that processes and forwards data to Elasticsearch or other storage systems.

  3. Kibana (K): A web-based data visualization and management tool that serves as the primary interface for Elastic SIEM. Kibana allows you to create dashboards, visualizations, and manage data within the Elastic Stack.

In addition to these components, Elastic Stack also includes Beats, which are lightweight data shippers that can be installed on endpoints to collect and send data to Elasticsearch or Logstash. Examples of Beats include Filebeat, Metricbeat, and Winlogbeat.

With the acquisition of Endgame, Elastic combined the capabilities of Beats into a single executable called Elastic Agent, which is now part of the Elastic Stack.

Elastic Agent and Fleet

Elastic Agent is a unified, extensible agent that can be installed on endpoints to collect and ship data to the Elastic Stack. The agent can be managed using Fleet, a centralized management platform accessible through Kibana. Fleet allows you to manage agent policies and integrations to monitor and collect data from various sources.

Elastic Stack Architecture

The Elastic Stack architecture consists of several components working together to provide a comprehensive SIEM solution. Key elements include:

  1. Elasticsearch: Stores and processes data, providing fast search and analytics capabilities.

  2. Logstash: Collects, processes, and forwards data to Elasticsearch or other storage systems.

  3. Kibana: Serves as the primary interface for Elastic SIEM, allowing users to create dashboards, visualizations, and manage data.

  4. Elastic Agent: Installed on endpoints, the Elastic Agent collects and ships data to Elasticsearch or Logstash.

  5. Fleet: Centralized management platform for Elastic Agents, accessible through Kibana.
