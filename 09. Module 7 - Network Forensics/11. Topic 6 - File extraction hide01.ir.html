Network Forensics > [[Topic 6: File extraction]]

In Wireshark, you have the ability to extract files from network captures, allowing you to save and analyze specific objects of interest. This feature is particularly useful when extracting files transferred over HTTP, SMB, and FTP protocols. The "Export Objects" dialog box provides a convenient interface.

To access the "Export Objects" dialog box, follow these steps:

  1. Open Wireshark and load the desired capture file or start a live capture.
  2. Navigate to the menu and select "File" > "Export Objects" > "HTTP" (or the relevant protocol).

The "Export Objects" dialog box will appear, listing the objects that can be exported based on the selected protocol. This list is automatically updated when captured in real-time, ensuring you have the latest objects available.

In the dialog box, you will find the following columns:

  1. Packet: Indicates the packet number in which the object was found. Note that multiple objects can be present in the same packet.
  2. Hostname: Displays the hostname of the server that sent the object.
  3. Content-Type: Specifies the type of content for the object.
  4. Size: Indicates the size of the object in bytes.
  5. Filename: Shows the filename assigned to the object. The naming convention varies depending on the protocol. For example, HTTP uses the final part of the URI, while IMF uses the subject of the email.

The "Export Objects" dialog box provides several options to refine your file extraction process:

  1. Text Filter: Filters and displays only objects containing the specified text string.
  2. Help: Opens the relevant section of the user's guide for further assistance.
  3. Save All: Saves all objects, including those not currently displayed. You will be prompted to select the directory or folder to save them.
  4. Close: Closes the dialog box without exporting any objects.
  5. Save: You can save the currently selected object using a specified filename. By default, the filename is taken from the filename column of the objects list.

By leveraging the file extraction capability in Wireshark, you can extract and analyze specific objects from network captures, providing deeper insights into the communication and data exchanged during network interactions.

In addition to Wireshark's built-in file extraction capabilities, you can utilize external tools to analyze extracted files further. One such tool is Binwalk, designed specifically for searching binary files and extracting embedded files and signatures.

Binwalk is a powerful command-line tool that performs file analysis and extraction based on file signatures, headers, and magic numbers. It can help identify hidden or embedded files within extracted objects, providing valuable insights during forensic investigations.
