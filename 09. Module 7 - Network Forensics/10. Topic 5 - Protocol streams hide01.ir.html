Network Forensics > [[Topic 5: Protocol streams]]

A protocol stream refers to the sequence of network packets associated with a specific protocol or communication session between network entities. It represents the data flow between the sender and receiver within a particular protocol.

Following Protocol Streams provides a convenient way to view protocols from an application layer perspective. Whether you're searching for specific information within a Telnet stream or trying to make sense of a data stream, Wireshark's ability to follow protocol streams is invaluable. By filtering to a particular stream, you can focus on the packets relevant to that stream.

To follow a specific stream, select a TCP, UDP, DCCP, TLS, HTTP, HTTP/2, QUIC, or SIP packet in the packet list and choose "Analyze Follow TCP Stream" from the menu, or use the context menu. Wireshark will apply the appropriate display filter and present a dialog box displaying the stream data.


The "Follow TCP Stream" dialog box in Wireshark displays the stream content in the order it appeared on the network. Non-printable characters are replaced with dots, and traffic from the client to the server is colored red, while traffic from the server to the client is colored blue. You can customize these colors in the preferences menu.


Follow the stream window - zoom in

The dialog box provides several actions to interact with the stream data. You can filter out the current stream data, print it, save it in different formats, go back to the previous display filter, or close the dialog box while keeping the current filter intact. You can also select the entire conversation or switch between client-to-server and server-to-client data.

Different Formats for Viewing Data Wireshark allows you to view the stream data in various formats, including ASCII, C Arrays, EBCDIC, HEX Dump, UTF-8, UTF-16, and YAML. Each format serves different purposes, such as importing data into your own C program, viewing binary data, or decoding data in different character encodings. You can select the desired format from the options provided.

Analyzing HTTP/2 and SIP Streams Wireshark offers specific dialog boxes for following HTTP/2 and SIP streams. Similar to the TCP stream dialog, you can filter and analyze HTTP/2 streams based on the stream index. For SIP calls, the filter is based on the sip.Call-ID field.
