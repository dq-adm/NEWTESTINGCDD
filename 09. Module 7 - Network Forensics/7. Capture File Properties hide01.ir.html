Network Forensics > [T4]: Capture File Properties

"Capture File Properties" window in Wireshark provides information about the current capture file, allowing analysts to gain insights into the captured network traffic. This window presents various details and statistics related to the capture file, offering a comprehensive overview for analysis.

"Capture File Properties" window - zoom in

  1. File: General information about the capture file, including its full path, size, cryptographic hashes, file format, and encapsulation.

  2. Time: Timestamps of the first and last packet in the file, along with the time difference between them. This information helps establish the capture duration and build a timeline to correlate network events with other artifacts.

  3. Capture: Information about the capture environment. Note that the pcapng format supports this feature, while pcap does not.

  4. Interfaces: Details about the capture interface(s) used during the capture, providing insights into the network devices involved.

  5. Statistics: A statistical summary of the capture file. If a display filter is applied, the "Captured" column displays values based on the filter, while the "Marked" column shows values for any marked packets. The "Displayed" column reflects the values corresponding to the packets shown in the display.

  6. Capture file comments: Some capture file formats, notably pcapng, allow text comments for the entire file. This section allows viewing and editing of such comments.

By utilizing the valuable insights provided by the "Capture File Properties" window, network forensic analysts can effectively examine and analyze capture files, extracting crucial information and uncovering meaningful patterns within the captured network traffic.
