PSXVIEW Plugin

PSXVIEW utilizes list walking and brute-forcing techniques to list discovered processes across seven different views: pslist, psscan, thrdproc, pspcid, csrss, session, and deskthrd. A 'True' value indicates that the process is visible using that method, and a 'False' means it's hidden.


psxview output - zoom in

 

By comparing the output, you can identify discrepancies and spot processes that were detected by one method but not the other. This plugin is beneficial in detecting rootkits and different types of malware that use hidden processes to evade detection.


psxview CCDrootkit.exe - (zoom in)


Let's have a look at how to use PSXVIEW in a typical investigation:

  1. The first step is to check the entries in the pslist column. What you see in PSLIST is what you see in Windows Task Manager. PSLIST extracts processes from the "PsActiveProcessHead" list. All PSLIST hidden processes (i.e., FALSE) are good candidates for further investigation. However, not every PSLIST hidden process should be assumed malicious. Some legit situations may result in a process not being detected using some methods (e.g., terminated processes). As you can see in the screenshot above, 'CCDrootkit.exe' is the only hidden process in PSLIST.
  2. Looking at the rest of the methods, you can see that the 'CCDrootkit.exe' process is hidden in all views except csrss and session.
  3. The next step is to record the address of the suspicious process (CCDrootkit.exe) identified in previous steps (0x0000000030584080). This is for further examination using PSINFO, which we will discuss later.

Note that some process-related plugins may display inaccurate information about process names and PIDs. The reason may be irrelevant data in the dump that match some plugin-related fingerprints (i.e., false positives).

To conclude, The psxview plugin makes finding hidden processes in a memory dump easier by combining multiple methods and presenting the result into one view, making the process more efficient and streamlined.
