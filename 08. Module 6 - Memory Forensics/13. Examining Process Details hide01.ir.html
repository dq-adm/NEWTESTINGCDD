PSINFO Plugin

In the previous section, you identified a suspicious hidden process using PSXVIEW. To investigate further, you will run the PSINFO plugin against the suspicious process.

 

The PSINFO plugin is a Volatility tool used to display detailed process information. It can show the process ID, parent process ID, user account, executable path, and start time, among other things. This information can help identify suspicious processes or trace the activities of a particular process. PSINFO is not included in Volatility by default, but you can download it separately here.

Let's see how we can use PSINFO to inspect the suspicious process 'CCDrootkit.'

psinfo output - zoom in

  1. The first step is to run PSINFO against the 'CCDrootkit' process offset/address you have from PSXVIEW.
  2. Process Information: as you can see, we have many helpful process details in one place, such as PID, parent process, creation time, and Command-Line. Checking the parent process and the process command line can shed some light and provide a better context. Note that PSINFO could not identify the parent process name. However, it recognized its PID 1944. Rerunning PSINFO against that parent process would be the following expected action.
  3. VAD and PEB comparison: This part retrieves process details from two locations, VAD (Virtual Address Descriptor) and PEB (Process Environment Block), to check for anomalies. Similar to the cross-view detection we have seen earlier, a mismatch increases the odds of being a malicious process.

    VAD (Virtual Address Descriptor) and PEB (Process Environment Block) are data structures used in the memory management of Windows operating systems. VADs track the memory allocated to a process, while PEBs contain information about the process, such as its loaded modules, command line arguments, environment variables, and more. While VADs and PEBs are related, they serve different purposes in memory management and contain different types of information.
  4. Similar Processes: other discovered processes with the same name.
  5. Suspicious Memory Regions: This part lists memory regions that contain malicious code. It can be helpful in the detection of process injection attempts.

One important thing to highlight is that PSINFO does not have to be used after PSXVIEW. You can use it anytime to have a detailed view of any process you suspect. Here is a recommended read by the PSINFO author for more use cases.
