Volatility2's Imageinfo Plugin

Volatility2 uses profiles for each operating system to understand the structure of the memory and to parse the data correctly. Without the correct profile, the analysis may not be accurate or complete. The 'imageinfo' plugin in Volatility2 is very helpful in identifying the operating system, service pack, and hardware architecture (32 or 64-bit) of the subject system.

Another important piece of information that the imageinfo plugin can provide is the time zone of the target system to establish a timeline of activity on the system, which can be crucial in identifying potential suspects and understanding the scope of the incident.

Finally, it can also be used to determine the hardware architecture of the target system (i.e., 32-bit or 64-bit) and the address of the KDBG, a data structure used for debugging purposes if the system crashes. It's also used to keep track of information about the system, like running processes. The KDBG is unique for each operating system and can fingerprint the target system.

imageinfo output - zoom in

 

  1. Suggested Profiles: The imageinfo plugin scans the entire memory dump file looking for KDBG (kernel's debugging structure) to determine the OS build. The reason for suggesting multiple profiles is that multiple OS builds may share the same KDBG. As you can see in the above screenshot, the tool suggested multiple profiles, all for Windows 10 but with different builds.
  2. Image date and time: refer to the time of the system when the memory dump was taken in the UTC timezone. The "Image local date and time" refers to the suspected system's local time (local timezone). In other words, both times are the same, but the first is in the UTC timezone, and the second is in the subject system's local time zone.
