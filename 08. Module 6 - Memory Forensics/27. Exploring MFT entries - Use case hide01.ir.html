Memory Forensics > [T5]: Exploring MFT entries: Use case

When you start the investigation of MFT records in memory, you will start by looking at common places where malicious files may reside. As you can see in the following image, we found a file called persist.ps1 in C:\Windows\Temp. 
 
Here are some top locations where malicious files may reside on Windows:
  • C:\Windows\Temp: This is a temporary folder where applications and programs store temporary files. Malware may use this folder to store and execute its malicious code.
  • Downloads: As the name suggests, this folder stores downloaded files. Malware can easily trick users into downloading and executing malicious files from this folder.
  • Program Files and Program Files (x86): contain files and programs installed on your system. Malware may install itself in these folders to hide among legitimate files.
  • System32 and SysWOW64: contain important system files for the operating system to run. Malware may infect these files to gain control over the system.
  • AppData: This folder contains application data and settings for each user. Malware may store and execute malicious code in this folder to evade detection.
  • Startup: Malware may add itself to the Startup folder to execute automatically when the system starts up.
  • Documents: This is another common location where users store important files. Malware may infect these files to spread to other systems or hold them for ransom.

mftparser (persist.ps1) - zoom in

  1. The MFT record physical offset of the file is located at 0x26c09c00 offset. The "File" attribute is the only attribute associated with this file. As you can see, the "in use" attribute is not there, meaning this file was deleted. The order of files in MFT is 109831 and has only one hard link, so you won't find any other copy of this file on the disk.
     
  2. From the timestamp in $STANDARD_INFORMATION, you can build a timeline and check when this file was created on the compromised machine.
     
  3. In the $FILE_NAME section of a file, which contains its name and path, you can identify timestomping techniques by comparing timestamps with the $STANDARD_INFORMATION section. Timestomping is a technique used by attackers to change file timestamps and avoid detection. If you want to learn more about how MAGNET AXIOM detects timestomping, you can follow this link.
     
  4. The $OBJECT_ID section contains the unique GUID assigned to the file and the GUID of the volume where the file resides. mftparser plugin already cross-references this information, so you won't need to do this manually.
     
  5. The last part of the output contains the value of the $DATA attribute, which contains the content of the PowerShell script. As you can see, the attacker created a script to create a value named "Chrome update" in the Run key, which points to the "CCDrootkit.exe" executable.

To make the process easier and faster, a recovery tool called Rstudio can scan the entire memory dump looking for MFT entries and representing the files like the Windows file explorer.
