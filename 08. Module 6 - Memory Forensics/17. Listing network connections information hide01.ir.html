Memory Forensics > [T3]: Listing network connections information

Connections - netscan plugin

Pool tags are four-byte identifiers the OS assigns to each object in memory to differentiate between different types of objects. Windows use those tags to track memory allocations in the kernel pool, allowing for easier debugging and memory usage analysis. Popular pool tags are:

  • Thds: used for thread objects.
  • Proc: used for process objects.
  • File: used for file objects.
  • TcpL, TcpE, and UdpA: used for network objects.

The Windows operating system uses the TcpL, TcpE, and UdpA pool tags to allocate memory for various network-related objects, such as network connections and sockets. These pool tags can be helpful in identifying network connections and activity in a memory dump.

The netscan Volatility plugin scans the memory for these pool tags. It extracts the associated data structures and network-relevant information (i.e., connections), including the source and destination IP addresses, port numbers, and other information.

Network connections have three different memory structures based on the protocol type (TCP or UDP) and the connection state.

  • UdpA: used for UDP sockets.
  • TcpL: used for listening TCP sockets.
  • TcpE: used for established TCP connections
The plugin parses all the network structures in memory images. The plugin works on memory images, using a brute force scan for the TcpL, TcpE, and UdpA pool tags. The output of this plugin is designed to imitate the netstat command output.

Let's have a quick look at the 'netscan' plugin output:

netscan - zoom in

  1. Offset(P): contains the offset (memory address) of the network connection structure in memory.
  2. Proto: protocol type (TCP or UDP) and IP version (v4 or v6).
  3. Local address: the IP address of the system. Possible formats you may see as IP addresses are:
    • IP Address:Port: This format indicates both the IP address of the local interface and the port number used for the connection.
    • [::]:Port: This format indicates that the connection is using an IPv6 address, represented by the "[::]" notation and the port number.
    • [::1]:Port: This format indicates that the connection is using the IPv6 loopback address, represented by the "[::1]" notation and the port number.
    • 0.0.0.0:Port: This format indicates that the connection is listening on all available interfaces on the local machine and the specified port.
  4. Foreign address: the remote IP address and port. Note that for UDP connections, you will not find entries in the foreign address column (i.e., *.*) since it's a connectionless protocol.
  5. State: present only for TCP connections since UDP is a stateless protocol. Possible values are: CLOSED, LISTENING, SYN_SENT, SYN_RCVD, ESTABLISHED, FIN_WAIT1, FIN_WAIT2, CLOSE_WAIT, CLOSING, LAST_ACK, TIME_WAIT, DELETE_TCP, which helps to identify the state of the TCP connection. Check out this article for the meaning of each state.
  6. PID and Owner: the PID and the name of the process associated with the network connection.
  7. Created: the connection (not the process) creation time.
