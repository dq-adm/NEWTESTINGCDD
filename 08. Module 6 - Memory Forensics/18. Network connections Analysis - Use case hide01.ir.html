Netscan investigation example


netscan - zoom in

Here is a real example of investigating a malicious process using netscan:

  1. The first thing that looks suspicious is seeing a process for an application not installed on the subject system, a common technique by attackers to hide the malicious process. Thus, it's essential to be aware of the applications installed on the system before starting your analysis. Context is everything! Seeing the 'teams.exe' process communicating with a public IP, although Microsoft Teams is not installed on the system, is very suspicious.  
    You can see that the "teams.exe" process was connected to "136.128.30.67" on port 4337. Unlike standard services like HTTP (port 80) or HTTPS (port 443), port 4337 is not typically associated with a specific service. Therefore, it's worth further examining this process using PSINFO to get a closer look at it.
     
  2. In the second last entry, you may have noticed that the plugin could not obtain the process ID, process name, and creation time. This is a common occurrence because sometimes the memory parts that contain that information may get overwritten by other applications or functions.
