Memory Forensics > [T2]: Processes listing

Several Volatility plugins can help you extract forensic artifacts from a memory dump. The first and most popular is PSLIST.

(1) Processes list: PSLIST

Windows keeps track of all the running processes in the "PsActiveProcessHead" list. This list is used to manage and monitor all the processes that are currently running on the system. Different system components and tools, like the Task Manager, can access the "PsActiveProcessHead" structure to get information about the processes.

The 'pslist' plugin analyzes memory dumps by inspecting the "PsActiveProcessHead" list in the memory dump. It lists all running processes, their name, unique identifiers (process ID), the process that started them (parent process ID), and the time they were started and terminated. With this information, you can better understand the context of the subject system.
 

pslist output - zoom in

  • Offset (V): the process address in memory.
  • Name: the name of the process.
  • PID: process ID.
  • PPID: parent process ID.
  • Thds: number of threads this process started.
  • Hnds: Processes use handles to interact with objects such as files, registry keys, and other processes. By analyzing the number and type of handles open by a process, a forensic investigator can gain insight into the resources that a process accessed.
  • Sess: when a user logs onto a Windows system, a new session is created for that user. All processes a user started will be associated with the user's session and run in the context of that session. It can provide valuable information like who initiated a process and the privilege level under which it is running.
  • Wow64: indicates whether a process is running as a 32-bit process on a 64-bit operating system.
  • Start: process start time used to determine the timeline of events and help reconstruct activities performed on the system. It can also assist in determining if a process was running before or after a specific event, such as a data breach or malware infection, which can help attribute actions to a particular process.
  • Exit: process termination time.

 

By looking at PSLIST output, you can follow these steps to screen the dump and identify anomalies quickly.

  • Identify processes with abnormal names.
  • Identify processes started when the incident was detected (or relatively close to time).
  • Check for processes singletons: a "singletons" process is a unique process in a memory dump that should only have one instance running (usually system core processes). If there is more than one process with that name, it can indicate suspicious behavior. Malware authors may name a process after a core Windows process to disguise it as legitimate and avoid detection. Spotting these "singletons" abnormalities can help identify malicious activity in a memory dump. There are some known processes as singletons:System
    • wininit.exe
    • lsass.exe
    • services.exe
    • spoolsv.exe
